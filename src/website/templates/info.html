How AiMo Works

Our movie recommender system is designed to deliver personalized movie recommendations based on your preferences and search queries.

System Architecture

  • Flask Backend: The backbone of our system, Flask manages user interactions and database operations.
  • TiDB Database: Stores user data, including languages, genres, and tuning preferences.
  • TiDB Vector Client: Efficiently searches and retrieves movies based on semantic similarity to your query.
  • Sentence Transformer Model: Converts text into vector embeddings for accurate similarity comparisons.
  • IMDbPy Cinemagoer: Fetches movie data, such as genres and languages, to enrich the recommendation process.
  • Environment Configuration: Ensures secure handling of sensitive data via environment variables.

How It Works

  1. Generating Embeddings:

    When you input a search query, it is converted into a vector embedding using our Sentence Transformer model. This vector is used to find similar movies in our database.

  2. Vector Store and Search:

    Your query embedding is compared against precomputed movie embeddings in the TiDB Vector Store, returning a list of potential movie recommendations.

  3. Filtering Recommendations:

    We filter these recommendations based on your stored preferences (genres and languages) to ensure that you only see movies that match your tastes.
